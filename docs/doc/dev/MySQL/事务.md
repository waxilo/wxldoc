

## 事务问题

| 问题       | 表现                           | 如何解决                                                   |
| ---------- | ------------------------------ | ---------------------------------------------------------- |
| 脏读       | 事务A能读到事务B未提交的修改。 | 读已提交通过每执行一次`select`生成一个读视图。             |
| 不可重复读 | 事务A能读到事务B已提交的修改。 | 可重复读通过第一次`select`生成唯一读视图，后续复用此视图。 |
| 幻读       | 事务A能读到事务B已提交的新增。 | 依赖读视图 + 临键锁实现。                                  |

[^读视图]: MVCC组件
