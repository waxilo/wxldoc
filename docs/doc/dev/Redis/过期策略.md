针对设置有过期时间的key，reids主要使用以下3中进行key删除：

| 过期策略 | 描述                                                                 | 优点             | 缺点           |
| ---- | ------------------------------------------------------------------ | -------------- | ------------ |
| 惰性删除 | 当每次访问key时，检查是否过期，过期则删除                                             | CPU友好          | 不访问的key无法被移除 |
| 定期删除 | redis每隔一段时间抽取一批设置了过期时间的key，<br />删除其中已过期的key；如果删除数量超过抽取数量25%，再次抽取。 | 折中方案           | 非即时删除        |
| 定时删除 | key绑定一个定时器，当key过期立即删除                                              | 删除及时<br />内存友好 | 需创建大量定时器     |
